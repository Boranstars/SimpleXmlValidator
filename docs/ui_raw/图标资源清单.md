# 图标资源清单（桶二：SVG + .qrc）

> 用途：本文件列出「贴近概念设计图」所需的全部图标素材，供素材收集使用。
> 目标格式统一为 **SVG**，通过 Qt 资源文件（`.qrc`）打包进可执行程序。
> 概念图见 `docs/ui_raw/概念设计图.jpg`；当前纯 QSS 版本已完成布局与配色，缺的就是这些图标。

## 1. 通用约定

- **格式**：SVG（矢量，避免多分辨率位图）。每图独立一个文件。
- **命名**：全小写、短横线连字符，例如 `open-xml.svg`。请严格使用下表「建议文件名」，接线代码将按此引用。
- **资源前缀**：统一放在 `:/icons/` 下（即 `.qrc` 里 `prefix="/icons"`）。代码引用形如 `QIcon(":/icons/open-xml.svg")`。
- **画布尺寸（viewBox）**：
  - 工具栏图标：`24×24`
  - 文件列表 / 表格级别 / 详情面板小图标：`16×16`
  - 窗口/应用图标：`256×256`（另附一份，用于 `setWindowIcon`）
- **颜色**：
  - **语义色固定**（状态/级别）：按下表指定颜色内置到 SVG（绿/红/橙/蓝/黄）。
  - **中性图标**（工具栏、浏览、文件类型）：建议使用单色 `#374151`，或用 `currentColor` 便于后续主题化。
- **风格**：扁平线性（line / outlined）风格，描边粗细约 `1.5–2px`，与概念图一致；避免写实、渐变、阴影。
- **内边距**：图形四周留约 10% 透明边距，避免贴边。

## 8. `.qrc` 与接线（收集完成后由开发接入）

素材齐全后，计划新增 `resources/icons.qrc`，结构如下：

```xml
<RCC>
  <qresource prefix="/icons">
    <file>app-logo.svg</file>
    <file>open-xml.svg</file>
    <file>open-xsd.svg</file>
    <file>validate.svg</file>
    <file>clear.svg</file>
    <file>file-xml.svg</file>
    <file>file-xsd.svg</file>
    <file>status-pass.svg</file>
    <file>status-fail.svg</file>
    <file>status-error.svg</file>
    <file>count-error.svg</file>
    <file>count-warning.svg</file>
    <file>elapsed.svg</file>
    <!-- 其余按需追加 -->
  </qresource>
</RCC>
```

- CMake 已开启 `CMAKE_AUTORCC`，将 `resources/icons.qrc` 加入 `SimpleXmlValidator` 目标源即可自动编译。
- SVG 图标需 Qt 的 svg 插件：接入时在 CMake 增加 `find_package(Qt5 COMPONENTS Svg ...)` 并链接 `Qt5::Svg`（否则 `QIcon(":/icons/x.svg")` 可能不渲染）。
- 代码接入点：工具栏 `QAction::setIcon`、窗口 `setWindowIcon`、列表 `QListWidgetItem::setIcon`、卡片图标 `QLabel::setPixmap`、表格级别列 `QTableWidgetItem::setIcon`。

## 9. 优先级建议

- **第一批（必需，直接对应已实现功能）**：`app-logo`、`open-xml`、`open-xsd`、`validate`、`clear`、`file-xml`、`file-xsd`、`status-pass/fail/error`、`count-error`、`count-warning`、`elapsed`。
- **第二批（可选/装饰或对应未纳入功能）**：`export-report`、`list-add/remove`、`level-*`、`pin`、`chevron-down`。
