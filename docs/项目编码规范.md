# 项目编码规范

本文档记录 SimpleXmlValidator 当前采用的补充编码约定。新增代码和修改现有代码时应遵守以下规则。

## 1. 头文件保护

- 所有头文件必须使用预处理宏定义进行重复包含保护。
- 不使用 `#pragma once` 代替宏定义保护。
- 宏名称应具有项目和文件路径相关的唯一性，例如：

```cpp
#ifndef SIMPLE_XML_VALIDATOR_CORE_VALIDATION_VALIDATION_RESULT_H
#define SIMPLE_XML_VALIDATOR_CORE_VALIDATION_VALIDATION_RESULT_H

// declarations

#endif
```

## 2. 优先使用 C++17 特性

- 如果 C++17 标准库接口或语言特性能简化代码、提高可读性或减少错误，应优先采用。
- 常见适用场景包括 `std::filesystem`、结构化绑定、`if constexpr`、`std::optional`、`std::string_view`、`std::variant` 和 `std::make_unique` 等。
- 使用新特性时仍需考虑项目的 Qt 5.12.x、Xerces-C++ 和 Windows/Linux 兼容性。
- 不为追求新特性而增加不必要的抽象；当传统写法更清晰或第三方接口不兼容时，以可维护性和正确性为准。
