# 软件功能描述

可接受xml文件和xsd文件，进行xml文件的校验。用户可以通过图形界面选择xml文件和xsd文件，点击校验按钮后，程序会调用xml解析库进行校验，并在界面上显示校验结果。
校验结果同时会输出到日志文件中，方便用户查看历史校验记录。软件还提供了基本的错误提示和帮助信息，帮助用户理解校验结果。

# 软件主要组成
- GUI界面模块：使用Qt框架实现，提供文件选择、校验按钮、结果显示等功能。
- XML解析模块：使用Xerces-C++或libxml2库实现，负责解析和校验XML文件。
- 日志模块：使用spdlog库实现，负责记录校验过程中的日志信息，包括错误信息和操作记录。有业务日志（xml解析结果）和系统日志（程序运行状态）。
- 测试模块：使用GTest框架实现，负责对各个模块进行单元测试和集成测试

# 软件主要功能模块

- 文件选择模块：支持选择 XML 文件和 XSD 文件，并检查文件路径有效性。
- XML 校验模块：调用 XML 解析库完成 XML 格式良好性检查和 XSD Schema 校验。
- 结果显示模块：在 GUI 界面中展示校验是否通过、错误行号、错误列号和错误原因。
- 日志记录模块：记录校验时间、文件路径、校验结果和错误详情。
- 错误处理模块：对文件不存在、格式错误、Schema 不匹配等异常情况给出提示。
- 测试模块：对 XML 校验逻辑、异常输入和典型样例进行单元测试。

# 设计思路
软件采用分层设计：

1. 界面层
   负责 XML/XSD 文件选择、校验按钮、结果表格显示和用户提示。

2. 业务逻辑层
   提供 XmlValidator 校验接口，接收 XML 文件路径和 XSD 文件路径，返回校验结果。

3. 第三方库封装层
   对 Xerces-C++ 的初始化、Parser 配置、Schema 加载、ErrorHandler 错误收集进行封装，避免 GUI 层直接依赖复杂的 Xerces 原始 API。

4. 日志模块
   记录校验时间、文件路径、校验状态和错误详情。
