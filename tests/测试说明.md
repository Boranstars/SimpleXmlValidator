# 测试说明

本文档记录测试范围、测试数据和验收方式。测试代码位于 `tests/`，XML/XSD 样例数据位于本目录。

## 1. 输入文件检查测试

测试对象为 `InputValidator::check`，仅验证文件路径和文件状态；不验证 XML 或 XSD 内容。

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 空路径 | 直接构造空 `std::filesystem::path` | 检查失败，提示路径为空，不返回有效路径 |
| 有效相对路径 | 测试运行时创建的非空临时文件 | 检查成功，返回绝对且词法规范化的路径，错误信息为空 |
| 有效绝对路径 | 测试运行时创建的非空临时文件 | 检查成功，返回绝对且词法规范化的路径，错误信息为空 |
| 中文和空格路径 | 测试运行时创建的中文、空格目录及文件 | 检查成功，正确处理规范化绝对路径 |
| 文件不存在 | 临时目录中的未创建路径 | 检查失败，提示文件不存在 |
| 路径为目录 | 测试运行时创建的临时目录 | 检查失败，提示不是普通文件 |
| 空文件 | 测试运行时创建的零字节文件 | 检查失败，提示文件为空 |
| 无读取权限 | 测试运行时创建并移除读取权限的文件 | 权限限制生效时检查失败；当前平台无法稳定模拟时跳过 |
| 非 XML/XSD 扩展名 | 测试运行时创建的非空 `.txt` 文件 | 检查成功，不因扩展名被拒绝 |
| 包含 `..` | 测试运行时创建的非空临时文件 | 检查成功，返回路径不保留 `..` |
| 多个连续分隔符 | 测试运行时创建的非空临时文件 | 检查成功，返回路径不保留冗余分隔符 |
| 文件名仅为 `.xml` | 测试运行时创建的非空隐藏式文件 | 检查成功，不因文件名形式被拒绝 |
| 特殊字符路径 | 测试运行时创建的包含跨平台安全特殊字符的文件 | 检查成功，返回规范化绝对路径 |
| 工作目录变化后的相对路径 | 切换至临时工作目录后创建的相对文件 | 检查成功，按切换后的工作目录返回规范化绝对路径 |
| 文件路径末尾包含分隔符 | POSIX 平台上的非空普通文件 | 检查失败，不将普通文件误判为目录 |

测试期间创建的文件和目录必须位于系统临时目录或测试工作目录，并在测试结束后清理。合法 XML/XSD 样例由后续 `XmlValidator` 测试复用 `tests/xml/` 和 `tests/xsd/` 中的文件。

极长路径受操作系统长路径配置影响，路径大小写差异受底层文件系统是否区分大小写影响，因此不纳入跨平台基础 GTest；需要时在对应目标平台补充专项测试。

## 2. Xerces 基础设施测试

测试对象为 `XercesString` 与 `XercesErrorHandler`。测试需要先初始化 `XercesRuntime`，但不涉及 XML 解析、XSD 加载或 Schema 校验。

### 2.1 `XercesString` 测试

| 场景 | 输入 | 预期结果 |
|---|---|---|
| 空 `XMLCh*` | `fromXMLCh(nullptr)` | 返回空 `std::string` |
| ASCII 往返 | `"schema error"` | `ScopedXMLCh` 非空，转换回来内容完全一致 |
| 空字符串语义 | 空 `std::string` | `ScopedXMLCh::get()` 非空，转换回来仍为空字符串 |
| 中文 UTF-8 往返 | UTF-8 中文文本 | 返回字节与原始 UTF-8 `std::string` 完全一致 |
| 混合文本 | 中文、ASCII、空格、数字和符号 | 往返后内容完全一致 |
| 空 C 字符串指针 | `nullptr` | `ScopedXMLCh::get()` 为 `nullptr` |
| 空 C 字符串 | `""` | `ScopedXMLCh::get()` 非空，转换结果为空字符串 |
| RAII 基本行为 | 重复创建和析构转码对象 | 不崩溃，转换结果稳定；内存泄漏由 Sanitizer 或 Valgrind 补充检查 |

Windows 平台额外编译三项专项回归：以固定 UTF-16 码元构造中文诊断并断言输出 UTF-8 字节、断言 UTF-8 输入转换为预期 UTF-16 码元，以及验证非 BMP 字符的 UTF-16 代理对。三项测试不依赖当前 ANSI 代码页，防止非 UTF-8 代码页下中文诊断乱码。

### 2.2 `XercesErrorHandler` 测试

| 场景 | 操作 | 预期结果 |
|---|---|---|
| `warning` 映射 | 注入一条 `SAXParseException` | 生成一条 `Warning` 错误，行列和消息正确 |
| `error` 映射 | 注入一条 `SAXParseException` | 生成一条 `Error` 错误，行列和消息正确 |
| `fatalError` 映射 | 注入一条 `SAXParseException` | 生成一条 `Fatal` 错误，行列和消息正确 |
| 多错误顺序 | 依次调用三种回调 | 返回列表保持发生顺序 |
| 行列保留 | 构造不同的行号、列号 | `ValidationError` 与输入异常一致 |
| 中文消息 | 使用包含中文文本的消息 | 错误描述字节与输入 UTF-8 文本一致；Windows 专项回归保证结果不受 ANSI 代码页影响 |
| `takeErrors()` 交付 | 收集多条后调用一次 | 返回完整列表 |
| 交付后状态 | 连续调用两次 `takeErrors()` | 第二次返回空列表 |
| `resetErrors()` | 收集后调用 `resetErrors()` | 后续 `takeErrors()` 返回空列表 |
| 重置后再收集 | `resetErrors()` 后继续回调 | 仅保留重置后的新错误 |

### 2.3 `XercesRuntime` 覆盖方式

- 不单独通过破坏进程级 Xerces 环境来模拟初始化失败。
- `XercesString` 和 `XercesErrorHandler` 测试夹具均检查运行时已成功初始化。
- 初始化失败时，相关测试应跳过并输出 `errorMessage()`，避免把环境问题误判为功能失败。

## 3. XML/XSD 校验流程测试

测试对象为 `XercesSchemaValidator` 与 `XmlValidator`。测试通过 `XercesTestFixture` 在测试套件内初始化一次 `XercesRuntime`；初始化失败时相关测试跳过，避免将运行环境问题误判为校验逻辑失败。

### 3.1 `XercesSchemaValidator` 测试

`XercesSchemaValidator` 测试直接验证 Xerces 基础设施层的 XSD Grammar 预加载、XSD 依赖解析和资源访问限制，不验证 `ValidationResult` 的最终状态映射。

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 本地相对 `include` | `tests/xsd/setting_check.xsd`、`tests/xsd/common_types.xsd` 与 A2 合法 XML 样例 | XSD 可成功预加载，返回 `Completed`，且不产生 XML 错误 |
| macOS UTF-8 本地依赖路径 | 仅 macOS：临时目录、主 XSD、`include` 依赖和 XML 文件名均包含中文、空格和 `é` | 返回 `Completed`，确保本地 XSD 相对依赖可通过 UTF-8 路径加载 |
| 缺失本地 `include` | 测试运行时创建引用不存在 `missing.xsd` 的 XSD | 返回 `Blocked`，错误列表为空，XML 解析不会开始 |
| XSD 语法错误 | 测试运行时创建结构不完整的 XSD | 返回 `Blocked`，错误列表为空，消息包含 XSD 加载失败说明 |
| 远程 Schema 依赖 | 测试运行时创建引用 `https://example.invalid/...` 的 XSD | 本地资源解析器拒绝访问网络资源，返回 `Blocked`，错误列表为空，消息包含拒绝原因 |

### 3.2 `XmlValidator` 测试

`XmlValidator` 测试验证输入检查、Schema 预加载和 XML 校验完成后的最终 `ValidationResult` 语义。合法与非法 fixture 使用参数化测试逐个执行；测试使用 `SIMPLE_XML_VALIDATOR_SOURCE_ROOT` 定位样例，不依赖 CTest 工作目录。

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 全部合法 XML | `tests/xml/valid/` 的 6 个样例，分别配合 `setting_check.xsd`、`mnt_rpt.xsd`、`check_report.xsd` | 每个样例均返回 `Valid`，错误列表为空；覆盖简单和复杂嵌套/重复元素结构 |
| 全部非法 XML | `tests/xml/invalid/` 的 24 个样例，分别配合对应 XSD | 每个样例均返回 `Invalid`，错误列表非空，且至少一条 `Error`/`Fatal` 诊断具有有效行号和列号 |
| XML 格式不良 | 测试运行时创建标签未闭合的 XML 和合法临时 XSD | 返回 `Invalid`；至少一条 `Fatal` 错误具有有效行号和列号，不误报为流程 `Failed` |
| 带 `targetNamespace` 的 XSD | 测试运行时创建带 `targetNamespace` 的 XSD 与匹配命名空间 XML | 返回 `Valid`，验证命名空间 Schema 的 Grammar 绑定分支 |
| 无效 XSD | 测试运行时创建结构不完整的 XSD 与普通 XML | 返回 `Failed`，错误列表为空，消息包含 XSD 加载失败说明；XSD 错误不进入 XML 错误表 |
| 输入路径异常 | 测试运行时传入不存在 XML 路径和有效 XSD | 返回 `Failed`，错误列表为空，消息包含 XML 输入检查失败说明 |

#### 关键专项断言

参数化测试保证全部 fixture 均被执行；以下专项断言保证关键类别不会因错误类型漂移而仅凭通用 `Invalid` 状态通过。

| 场景 | 样例 | 预期结果 |
|---|---|---|
| 必填元素缺失 | A2 缺少 `QueryTime`、A3 缺少 `Base`、A4 缺少 `System` | 返回 `Invalid`；错误信息包含对应元素名，且存在可定位的 `Error`/`Fatal` 诊断 |
| 必填属性缺失 | A2 缺少 `appname`、A4 缺少 `DeviceId` | 返回 `Invalid`；错误信息包含对应属性名，且存在可定位的 `Error`/`Fatal` 诊断 |
| 多错误收集 | A2、A3、A4 的 `multiple_errors` 样例 | 返回 `Invalid`；每个样例至少保留两条诊断，不锁定 Xerces 的精确错误总数 |

## 4. 日志模块与接入测试

测试对象为 `LogManager`、`LogModule`、`logValidationRecord` 以及 `XmlValidator` 的日志接入。测试仅通过项目日志接口和最终文本文件验证行为，不直接依赖或断言 spdlog 原始 API。所有日志文件均写入测试创建的临时目录，并在测试结束后清理。

### 4.1 日志模块测试

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 双通道与模块级别 | 临时目录、System/Validation 两个模块句柄 | 两类消息分别写入 `system_runtime.log` 与 `xml_validation.log`；内容包含模块名、级别和消息，不串写到另一通道 |
| 完整 XML 校验记录 | 构造含 Warning、Error、Fatal 的 `ValidationResult`，使用中文及空格路径 | XML 校验日志包含 XML/XSD 路径、状态、错误数量、说明、每条错误的级别、行号、列号和描述 |
| 单通道失败降级 | 预先创建与校验日志文件同名的目录，使 Validation 通道无法打开 | System 通道保持可用且能写入；Validation 通道返回空操作句柄，写入调用不抛异常 |
| 全部日志不可用降级 | 将日志目录设置为“普通文件的子路径” | 两个通道均不可用；申请模块、写入日志和写校验记录均不抛异常 |
| 长文本调用 | 使用较长模块名和消息文本 | 日志接口可正常调用，文本完整写入；覆盖 `std::string_view` 接口的调用路径 |
| 双通道日志滚动 | 临时目录，开启滚动并设置较小单文件上限和有限历史数量 | System 和 Validation 两个通道均产生历史文件；每个通道的历史文件数恰为配置上限，当前文件保留最后写入的标识消息 |
| 零值滚动配置 | 滚动开启，单文件大小和历史文件数量均设为 `0` | 内部采用安全下限；写入后当前文件和一个历史文件存在，最新消息可读，不因配置异常中断日志调用 |

### 4.2 `XmlValidator` 日志接入测试

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 合法 XML 日志 | A2 合法 XML 与对应 XSD，使用可用日志目录 | 返回 `Valid`；系统日志包含开始和结束状态摘要；XML 校验日志包含 `Valid`、路径和零错误记录 |
| 非法 XML 日志 | A2 缺少必填属性 XML 与对应 XSD | 返回 `Invalid`；XML 校验日志保留错误数量和实际错误描述；系统日志只记录状态摘要，不重复错误明细 |
| 日志不可用不改变校验结果 | 合法、非法和输入路径异常场景，分别使用无 Logger 与不可用 Logger | 两次调用的状态、消息、错误数量和错误明细保持一致；日志不可用不改变 `ValidationResult` |

`main.cpp` 的程序启动、退出与 Xerces 生命周期记录不在本阶段 GTest 中驱动 Qt 事件循环验证；该部分保留为无界面启动冒烟检查，并在 GUI 阶段补充端到端验证。

## 5. GUI 展示映射测试

测试对象为 `ValidationResultPresenter`。该类只负责将核心层 `ValidationResult` 转换为不依赖 Qt 的 `PresentedResult`，因此使用 GTest 验证展示状态，不创建窗口、不驱动文件对话框或弹窗。Presenter 源文件仅编入测试目标，测试目标继续只链接 `SimpleXmlValidatorCore` 与 GTest，不链接 Qt。

| 场景 | 测试数据来源 | 预期结果 |
|---|---|---|
| 初始或重选文件状态 | 调用 `initial()` | 不显示提示条、错误表格或阻断性弹窗；错误数量、表格行和文本均为空/零值 |
| 合法结果 | 构造含附加错误数据的 `Valid` 结果 | 映射为绿色“XML 通过校验”；不显示错误表格或阻断性弹窗，不渲染附加错误数据 |
| 普通校验失败 | 构造包含 Warning、Error、Fatal 的 `Invalid` 结果 | 映射为红色“XML 未通过校验”；显示原始错误总数和表格；级别、行号、列号、描述均完整转换 |
| 位置未知的错误 | 构造行号、列号均为 `0` 的 `Invalid` 错误 | 表格行保留字符串 `"0"`，不伪造定位信息 |
| 完全重复的错误 | 构造相同级别、行列和描述的多条错误 | 表格按首次出现顺序合并为一行，`occurrences` 为真实重复次数；总错误数仍按合并前计算 |
| 不同错误不得误合并 | 构造仅级别、行号、列号或描述之一不同的错误 | 各诊断保留独立表格行，且顺序与首次出现顺序一致 |
| 大量错误截断 | 构造超过 `kMaxErrorRows` 的不同错误，并混入已展示错误的重复项 | 表格行数固定为上限，`truncated` 为真；已展示类别仍能累积次数，总错误数保留全部原始诊断数量 |
| 阻断性失败 | 构造带消息和空消息的 `Failed` 结果 | 不显示普通提示或错误表格；带消息时原样作为弹窗文本，空消息时使用明确的默认阻断提示 |

文件对话框取消、独立拖放、按钮启停、同步校验调用和实际 `QMessageBox` 展示属于 `MainWindow` 的 Qt 集成交互。本阶段不为这些行为引入 QtTest 或额外依赖注入；它们在 GUI 手工冒烟中按 `docs/模块实施计划.md` 的阶段 5 清单验收。
